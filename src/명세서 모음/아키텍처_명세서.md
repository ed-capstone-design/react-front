# 아키텍처 명세서

- **시스템 구조 다이어그램 설명**
  - [클라이언트(React)] ⇄ [API 서버(REST, WebSocket)] ⇄ [DB]
- **주요 모듈/컴포넌트 역할**
  - 프론트엔드: UI, 상태 관리, API 연동, 실시간 데이터 처리
  - 백엔드: 인증, 비즈니스 로직, 데이터 제공, WebSocket 브로커
  - DB: 데이터 저장, 조회, 관계 관리
- **요청/데이터 흐름**
  1. 사용자가 UI에서 요청
  2. 프론트엔드가 API 서버에 HTTP/WebSocket 요청
  3. 백엔드가 DB와 통신 후 응답 반환
  4. 프론트엔드가 데이터 렌더링
- **배포 구조**
  - 프론트엔드: 정적 호스팅(예: Netlify, S3)
  - 백엔드: 클라우드 서버/컨테이너
  - DB: 클라우드 DB 서비스
